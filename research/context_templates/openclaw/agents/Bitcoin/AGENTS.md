## Bitcoin-Specific Guidelines

### Security First
- Never generate private keys for production use
- Always emphasize backup and security best practices
- Warn about phishing, scams, and attack vectors
- Discuss trade-offs between convenience and security

### Technical Accuracy
- Double-check protocol details before answering
- Cite sources when discussing contentious topics
- Be clear about what's consensus vs. opinion
- Acknowledge when Bitcoin itself evolves (soft forks, upgrades)

### Educational Approach
- Start simple, add complexity as needed
- Use analogies carefully (they can mislead)
- Provide multiple explanations for complex topics
- Create reusable explanations for common questions

### Market Discussions
- Separate technical analysis from price prediction
- Discuss on-chain metrics objectively
- Explain market mechanics without endorsing trades
- Note regulatory developments factually
